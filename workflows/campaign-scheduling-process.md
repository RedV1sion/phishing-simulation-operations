## Campaign Planning & Scheduling Workflow

# Overview

This document outlines the enterprise phishing campaign scheduling process used to plan, configure, execute, monitor, and review phishing simulation campaigns within a Microsoft 365 enterprise environment.

The process supports:

- Security awareness operations
- User risk reduction
- Phishing susceptibility testing
- Campaign coordination
- Remediation management
- Security awareness maturity improvement

# Campaign Scheduling Workflow

```text
Campaign Objective Defined
            ↓
Target User Groups Identified
            ↓
Phishing Template Selected
            ↓
Landing Page Configured
            ↓
Campaign Difficulty Assigned
            ↓

            ↓
```

# 6. Campaign Scheduling Strategy

Campaigns are scheduled to:

- Avoid predictable patterns
- Simulate realistic phishing timing
- Test multiple business hours
- Evaluate remote workforce exposure
- Prevent user anticipation

---

# Example Scheduling Practices

| Scheduling Strategy | Purpose |
|---|---|
| Randomized Send Times | Increase realism |
| Business Hour Delivery | Simulate real attacks |
| Quarterly Campaign Rotation | Maintain awareness |
| Department-Specific Scheduling | Target high-risk users |
| Recurring Monthly Campaigns | Continuous awareness |

---

Campaigns are categorized by difficulty level:

| Difficulty | Description |
|---|---|

# 8. Campaign Approval Process

Before launch, campaigns are reviewed for:

- Technical accuracy
- Organizational appropriateness
- Awareness objectives
- Reporting configuration
- Remediation workflow integration

Approval is coordinated by:

- Security Operations Team
- Security Awareness Coordinator
- IT Management (where applicable)
