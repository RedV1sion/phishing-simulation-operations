# User Escalation Process

## Enterprise Phishing Operations
## High-Risk User Escalation Workflow

---

# Overview

This document outlines the enterprise user escalation process used to identify, review, and escalate high-risk users demonstrating repeated phishing-related security risks within a Microsoft 365 enterprise environment.

The escalation process supports:

- Enterprise risk reduction
- Repeat offender management
- User accountability
- Security awareness enforcement
- Management visibility
- Security operations coordination

---

# Environment Overview

| Item | Details |
|---|---|
| Email Platform | Microsoft 365 |
| Security Awareness Platform | KnowBe4 |
| Identity Platform | Microsoft Entra ID |
| SIEM Platform | Microsoft Sentinel |
| User Base | 120+ Users |
| Environment Type | Hybrid Workforce |

---

# Purpose

The purpose of the escalation process is to:

- Identify high-risk users
- Reduce organizational phishing risk
- Escalate repeated risky behavior
- Improve security awareness accountability
- Protect enterprise systems and sensitive data

---

# User Escalation Workflow

```text
User Fails Phishing Simulation
            ↓
User Risk Score Updated
            ↓
Repeat Offender Threshold Reached
            ↓
Additional Remediation Assigned
            ↓
User Behavior Reviewed by Security Team
            ↓
Risk Severity Evaluated
            ↓
Management Escalation Initiated
            ↓
Security Coaching & Awareness Reinforcement
            ↓
Additional Monitoring & Follow-Up Simulations
            ↓
Risk Reassessment Completed
```

---

# Escalation Triggers

Users may be escalated when they demonstrate:

- Multiple phishing failures
- Credential submission attempts
- Repeated high-risk behavior
- Failure to complete remediation training
- High-risk interactions involving financial or privileged accounts
- Excessive susceptibility to phishing simulations

---

# Escalation Thresholds

| Threshold | Escalation Action |
|---|---|
| 2 Failures | Additional awareness training |
| 3 Failures | Targeted phishing simulations |
| 4+ Failures | Management review |
| Repeated credential submissions | Immediate escalation |
| Failure to complete remediation | Escalation review |

---

# High-Risk User Categories

| Risk Category | Description |
|---|---|
| Medium Risk | Occasional phishing interaction |
| High Risk | Multiple phishing failures |
| Critical Risk | Repeated credential submission or repeated failures |

# Common High-Risk Behaviors

The following behaviors commonly trigger escalation reviews:

- Repeated phishing link clicks
- Credential submission attempts
- Failure to report suspicious emails
- Interaction with advanced phishing campaigns
- Trust in urgency-based phishing messages
- High interaction with Microsoft impersonation attacks

# Security Team Review Process

Before escalation, the Security Operations Team reviews:

- Campaign interaction history
- User risk score
- Remediation completion status
- Repeat offender history
- Department-specific risk
- Privileged access level
- Potential business impact

---

# Management Escalation Process

Escalations may involve:

- User’s direct manager
- Department leadership
- IT management
- Security awareness coordinator
- Human Resources (if necessary)

- Improve user awareness

- Reduce organizational phishing risk
